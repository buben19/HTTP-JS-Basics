String

String

Data type for holding text

Should be (and is) immutable

Basically, just sequence of characters

JS String has also some methods for common operations

JS supports regular expressions (regex)

Immutable object

Once initialized, data can't be altered

Its best practise, that object which represents data container is immutable

Package design pattern

There are some exceptions

Immutable object

let s = "Hello world";
console.log(s.toUpperCase()); // "HELLO WORLD"
console.log(s); // Hello world

String creation

let s1 = "Lorem ipsum dolor sit amet";
let s2 = 'Lorem ipsum dolor sit amet';
let s3 = `Lorem ipsum dolor sit amet`;
let s4 = new String("Lorem ipsum dolor sit amet");

String object vs primitives

let s1 = "Hello world";
let s2 = "Hello world";
s1 === s2; // true

let s3 = new String("Hello world");
let s4 = new String("Hello world");
s3 === s4; // false

String object vs primitives

let s1 = "Hello world";
typeof s1; // "string"

let s2 = new String("Hello world");
typeof s2; // "object"

let s3 = s2.toString();
typeof s3; // "string"

Long Strings

let s1 = "Lorem ipsum " +
  "dolor sit amet";

let s2 "Lorem ipsum \
dolor sit amet"

Escape sequences

Escape sequences

let s1 = "Say \"Hello\"";
let s2 = 'Say "Hello"';

Concatenation

let s1 = "Hello" + " world";
let s2 = "Hello".concat(" world");

let s3 = "Hello";
s3 += " world";

String formatting?

let count = 5;
let s1 = count + " apples"; // "5 apples"
let s2 = `${count} apples`; // "5 apples"

String operations

let s = "Lorem ipsum dolor sit amet";
s.length; // 26
s.toUpperCase(); // "LOREM IPSUM DOLOR SIT AMET"
s.substring(5); // " ipsum dolor sit amet"
s.substring(0, 5); // "Lorem"
s.replace("o", "a"); // "Larem ipsum dolor sit amet"
s.replaceAll("o", "a"); // "Larem ipsum dalar sit amet"
s.startsWith("Lorem"); // true
s.endsWith("amet"); // true
s.split(" "); // ["Lorem", "ipsum", "dolor", "sit", "amet"]
s.charAt(0); // "L"
s[0]; // "L"

Convert to other data types

Number.parseInt("1"); // 1
Number.parseInt("3.14"); // 3
Number.parseFloat("3.14"); // 3.14
+"3.14"; // 3.14
